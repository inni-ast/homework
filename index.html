Список тегов

abbr
Тег <abbr> указывает, что последовательность символов является аббревиатурой. Aтрибут title дает расшифровку сокращения, что позволяет понимать аббревиатуру тем людям, которые с ней не знакомы.
address
Тег предназначен для хранения информации об авторе и может включать в себя любые элементы HTML вроде ссылок, текста. Выделяется курсивом.
aside
Блок сбоку от контента для размещения рубрик, ссылок на архив, меток и другой информации. Он, как правило, называется «сайдбар» или «боковая панель».
code
Фрагмент кода в тексте. Можно использовать для команд, технических терминов и др. По умолчанию используется шрифт Monospace.
caption
Предназначен для создания заголовка к таблице и может размещаться только внутри контейнера <table>, причем сразу после открывающего тега. Такой заголовок представляет собой текст, по умолчанию отображаемый перед таблицей и описывающий ее содержание.
cite
Помечает текст как цитату или сноску на другой материал.Отображается курсивом.
fieldset
Предназначен для группирования элементов формы. Это облегчает работу с формами, содержащими большое число данных. Например, один блок может быть предназначен для ввода текстовой информации, а другой — для флажков. Браузеры для повышения наглядности отображают результат использования данного тега в виде рамки. Ее вид зависит от операционной системы, а также используемого браузера
figcaption
Содержит описание для тега <figure> и вложен в него. Тег должен быть первым или последним элементом в группе <figure>.
legend
Тег применяется для создания заголовка группы элементов формы, которая определяется с помощью тега <fieldset>. Группа элементов обозначается в браузере с помощью рамки, а текст, который располагается внутри тега, встраивается в эту рамку.
mark
Тег помечает текст как выделенный. Такой текст ничем не отличается от обычного. Это смысловое выделение.
output
Определяет область в которую выводится информация, преимущественно с помощью скриптов.
optgroup
Тег представляет собой контейнер, внутри которого располагаются теги <option> объединенные в одну группу. Особенностью тега - он не выделяется как обычный элемент списка, а акцентируется с помощью жирного начертания, а все элементы, входящие в этот контейнер, смещаются вправо от своего исходного положения.
option
Тег формы, определяет отдельные пункты списка, создаваемого с помощью контейнера <select>.
picture
Служит контейнером для одного или более элементов <source> и одного элемента <img> для обеспечения оптимальной версии изображения для различных размеров экрана.
progress
Отображает индикатор, показывающий ход выполнения задачи, обычно отображаемый в виде прогресс бара (индикатора выполнения).
section
Представляет собой автономный раздел, который не может быть представлен более точным по семантике элементом — внутри HTML-документа. Как правило, но не всегда, разделы имеют заголовок. На странице их может быть несколько.
samp
Используется для отображения текста, который является результатом вывода компьютерной программы или скрипта. Браузеры обычно отображают текст с помощью моноширинного шрифта.
summary
Указывает заголовок для тега <details>, по которому можно щелкать для разворачивания/сворачивания информации. Тег должен идти первым внутри <details>.
tfoot
Предназначен для хранения одной или нескольких строк, которые представлены внизу таблицы.
th
Предназначен для создания одной ячейки таблицы, которая обозначается как заголовочная. Текст в такой ячейке отображается браузером обычно жирным шрифтом и выравнивается по центру.
thead
Предназначен для хранения одной или нескольких строк, которые представлены вверху таблицы. Допустимо использовать не более одного элемента в пределах одной таблицы, и он должен идти сразу после тега <table>.
time
Тег для указания даты, времени или обоих значений. Может указываться непосредственно внутри контейнера <time>, либо задаваться через атрибут datetime.
track
Позволяет авторам указать текстовую дорожку для медийных элементов <audio> и <video>. Такая дорожка обычно содержит субтитры на разных языках, комментарии, заголовки и др.
meter
Используется для вывода значения в некотором известном диапазоне. Используется преимущественно для отображения числовых значений, например, количества результатов поиска, объема жидкости, давления и др.
nav
Задает навигацию по сайту. Если на странице несколько блоков ссылок, то в тег обычно помещают приоритетные ссылки. Также допустимо использовать несколько тегов <nav> в документе. Запрещается вкладывать тег внутрь <address>.
menu
Экспериментальная технология. Теперь он выступает контейнером для тега <command> и создания меню. Он включает как меню-списки, которые могут отображаться в верхней части экрана, так и контекстные меню, например, такие, что могут появиться под кнопкой после нажатия.
menuitem
Экспериментальная технология. Задаёт команду, которую пользователь может вызывать через контекстное меню. Команда выглядит как текстовая метка, а также дополнительно может содержать небольшую картинку. Сама команда связана с программой на JavaScript.
keygen
Не ипользуется.
main
Предназначен для основного содержимого документа. Содержимое должно быть уникальным и не включать типовые блоки вроде шапки сайта, подвала, навигации, боковой панели, формы поиска и т. п. Документ не должен иметь более одного элемента <main> у которого не указан атрибут hidden.
embed
Bспользуется для загрузки и отображения объектов (например, видеофайлов, флэш-роликов, некоторых звуковых файлов и т.д.), которые исходно браузер не понимает. Как правило, такие объекты требуют подключения к браузеру специального модуля, который называется плагин, или запуска вспомогательной программы.
h1
Заголовок, может быть использован на странице только один раз.
article
Задает содержание сайта вроде новости, статьи, записи блога, форума или др.
